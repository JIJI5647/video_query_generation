PROMPT VERSION: omni_caption_prompt_unified_v1

You are generating UNSTRUCTURED multimodal captions for short video segments.

You are given one or more short video clips (with audio), each one temporal
segment of a longer video, provided IN ORDER. Watch and listen to every clip,
then caption EVERY clip — one caption per clip, no skipping and no merging.

CLIP -> SEGMENT MAPPING (the clips are provided in THIS order):
{segment_list}

The first video provided is Clip 1, the second is Clip 2, and so on. Use this
mapping to attach the correct segment_id and time_range to each caption.

Return ONLY a valid JSON array — one object per clip, in clip order. No markdown
fences, no extra text. Even for a single clip, return an array with one object.

Each caption object describes ONLY its own clip and must follow this structure
exactly (echo the segment_id and time_range from the mapping):
{
  "segment_id": "<the clip's segment_id from the mapping>",
  "time_range": [<start>, <end>],
  "visual": "<VISUAL prose, at most 3 sentences: the people (appearance,
    position, visibility), their observable actions, the scene, notable
    objects/interactions, and observable facial cues, body cues, posture,
    gestures and gaze>",
  "audio": "<AUDIO prose, at most 2 sentences: how the audio sounds — tone of
    voice, shouting, crying, laughing, heavy breathing, silence, unclear
    audio, off-screen voice. NOT a transcript; do not quote words. Leave empty
    if there is no audible cue>",
  "confidence": "<high | medium | low>",
  "evidence_strength": "<clear | ambiguous | weak>"
}

RULES (apply to every caption):

The caption must describe only the current clip.

"visual" is plain prose, HARD LIMIT 3 sentences. Do not use internal person IDs
such as person_1 or person_2 — use natural descriptions such as "the woman in a
white shirt" or "the man in a suit". Do not mention segment_id in
natural-language fields.

"audio" is plain prose, HARD LIMIT 2 sentences. Do not generate transcript
text — describe how the audio sounds, not the exact spoken words.

Do NOT name or infer any emotion anywhere in the caption. This is an
observation-only caption: describe what is visible and audible, never what the
person feels. Emotion is decided in a later, separate stage. If a cue is not
clearly visible or audible, write "unclear" / "not clearly visible" / "not
audible", and set evidence_strength to "weak"/"ambiguous" when cues are faint.

confidence must be one of: high | medium | low
evidence_strength must be one of: clear | ambiguous | weak

Output exactly one object per clip, in clip order, as a JSON array:
[
  { "segment_id": "...", "time_range": [..], "visual": "...", "audio": "...", "confidence": "...", "evidence_strength": "..." },
  ...
]
