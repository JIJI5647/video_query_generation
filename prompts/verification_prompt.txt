TASK:
Watch the provided video clip(s) carefully. You may be given ONE clip or SEVERAL clips:
when several are provided they are CONSECUTIVE segments of the same moment, in order —
treat them together as one continuous span of video. Then verify whether each query is
suitable for human annotation of that span.

You are given ONLY each query's id and its query_text. You are NOT given any caption,
grounding description, expected evidence, or query type — judge each query purely from
its own text together with what you observe in the video.

You judge every query against THREE criteria. A query is suitable only if it passes all
three.

VIDEO ID: {video_id}
Round Index: {round_index}

CRITERIA:

1. RELEVANCE — Is the query, OR the moment it points to, about one of the eight target
emotions?

The pipeline targets EXACTLY eight emotions (the same fixed set used to label captions):

  angry, excited, fear, sad, surprised, frustrated, happy, disappointed.

This is a loose gate — it is satisfied if EITHER the query OR the clip involves one of
the eight. Mark is_emotion_relevant = true if EITHER of the following holds:

* the QUERY concerns at least one of the eight — by naming the emotional state ("appears
  frightened", "looks happy"), by an action/interaction/audio event that expresses one
  (shouting in anger, crying in sadness, laughing in happiness, flinching in fear), or by
  an observable cue indicating one (widened eyes → surprise/fear, trembling voice →
  fear/sadness, clenched jaw → anger, ...); OR
* the MOMENT the query points to DISPLAYS at least one of the eight in the clip (visible
  or audible), even if the query itself is phrased as a neutral action.

Mark is_emotion_relevant = false ONLY when NEITHER holds — the query does not concern any
of the eight AND the moment it points to shows none of them (a purely neutral moment:
plain action, object, location, entrance/exit, movement, count, or identification). This
mirrors the caption labels "neutral" / "unrelevant".

This is only a loose in-domain gate: it asks whether one of the eight emotions is in play
(in the query or in the clip). Whether the query's SPECIFIC emotional claim is actually
correct is decided in Answerability below, not here.

2. ANSWERABILITY — Is the query supported by, and locatable in, the video clip(s)?

This is where you decide whether the query actually holds up against what you see and
hear. Watch the clip(s) and check whether a human annotator could locate the moment AND
whether the claim in the query is true of the clip(s). When several clips are given,
consider the moment present if it occurs anywhere across them.

PASS only if:

* the moment/behavior the query describes is actually present in the clip;
* there is visible or audible evidence supporting the query (for an emotional claim, the
  emotion or its observable cue is genuinely present — the query's emotion matches what
  the clip shows);
* the evidence is clear enough that careful annotators would agree on approximately where
  the relevant moment occurs.

FAIL if:

* the query cannot be answered from the clip, or references content not present in it;
* the query depends on hallucinated details;
* the emotion or behavior claimed by the query does NOT match the clip (e.g. the query
  says "appears frightened" but the clip shows a calm, neutral, or amused person);
* the relevant moment is too unclear, too brief, or too ambiguous to locate reliably;
* the query asks for genuinely unobservable mental content — what a person intends,
  knows, realizes, remembers, decides, wants, or understands — that cannot be read off
  observable behavior;
* the query requires plot knowledge, character identity, movie background, or other
  information outside the clip.

Important:

* Perceived emotional states ARE observable and ARE answerable when the clip shows
  visible or audible cues for them. Do not penalize a query just for naming an emotion
  ("appears upset", "looks frightened", "seems amused") — judge whether that emotion is
  actually supported by the clip.
* Only genuine mind-reading (intends / knows / realizes / remembers / decides / wants /
  understands) and reliance on outside knowledge make a query unanswerable on the
  "observable evidence" ground.

3. QUERY QUALITY — Is the query text itself well-formed for annotation?

This criterion is about WORDING ONLY. Do not re-judge emotion correctness or
observability here (that is Answerability). PASS only if the query_text satisfies ALL of:

* English: written entirely in English, with no non-English script or untranslated
  foreign dialogue.
* No proper nouns: no character, actor, celebrity, place, organization, or brand names
  (e.g. "Jack", "Rose", "Starbucks"). Use visual descriptions such as "the man in the
  dark jacket". Generic descriptors ("the man", "the woman", "the child", "the doctor",
  "the nurse") and visually grounded role/kinship terms (doctor, judge, mother, father,
  husband, wife, boy, girl) are allowed and are NOT proper nouns.
* Clear and unambiguous: one clear question, no confusing, contradictory, or overly
  complex phrasing.
* Not too broad: targets a specific moment or segment, not the entire video.
* Not repetitive: does not substantially overlap with another query in the same set.
* No explicit timestamps or time ranges in the query_text.

Hedging note: emotion wording may be hedged ("appears", "looks", "seems", "sounds") or
unhedged — hedging is an optional style choice and is NEVER, by itself, a reason to fail
or revise a query.

DECISION DEFINITIONS:

* pass: Relevance, Answerability, and Query Quality all pass. Ready for annotation as-is.
* revise: Relevance and Answerability pass, but Query Quality has a fixable wording issue
  (non-English text, an actual proper noun, vague person reference, slightly broad
  wording, minor overlap, or minor ambiguity). The emotional grounding is still valid.
* fail: Relevance fails, OR Answerability fails, OR Query Quality is broken beyond a
  simple rewrite.

VERIFICATION CHECKLIST, apply to each query:

Criterion 1 — Relevance:

* is_emotion_relevant: Does the QUERY concern one of the eight target emotions (angry,
  excited, fear, sad, surprised, frustrated, happy, disappointed), OR does the MOMENT it
  points to DISPLAY one of the eight in the clip? Mark true if either holds; mark false
  only when neither the query nor the moment involves any of the eight (mirrors caption

Criterion 2 — Answerability:

* is_answerable_from_video: Can a human annotator locate a specific moment from this query
  using visible or audible evidence in the clip, AND does the claim (including any
  emotion) actually match what the clip shows?
* is_grounded_in_observable_evidence: Does the query stay within observable evidence,
  rather than requiring genuine mind-reading (intends/knows/realizes/remembers/decides/
  wants/understands) or outside knowledge? Naming a perceived emotion is allowed.
* has_hallucination: Does the query reference content NOT present in the clip? True is bad.

Criterion 3 — Query Quality (wording only):

* is_english_only: Is query_text fully in English, with no untranslated foreign dialogue?
* avoids_proper_nouns: Does query_text avoid actual names of characters, actors,
  celebrities, places, organizations, and brands? Generic descriptors are allowed.
* is_clear_and_unambiguous: Is the query a single, clearly worded question?
* is_not_too_broad: Does it target a specific moment or segment rather than the whole video?
* is_not_repetitive: Is it distinct from the other queries in the set?
* no_timestamp_in_query_text: Does query_text contain no explicit timestamps or time ranges?

CRITERION ROLL-UPS:

* relevance_pass = is_emotion_relevant
* answerability_pass = is_answerable_from_video AND is_grounded_in_observable_evidence AND (NOT has_hallucination)
* query_quality_pass = is_english_only AND avoids_proper_nouns AND is_clear_and_unambiguous AND is_not_too_broad AND is_not_repetitive AND no_timestamp_in_query_text

FINAL DECISION LOGIC:

* If relevance_pass is false, decision must be "fail".
* Else if answerability_pass is false, decision must be "fail".
* Else if query_quality_pass is true, decision must be "pass".
* Else if query_quality_pass is false but the issue is fixable by rewriting the query_text
  while keeping the same emotional grounding, decision must be "revise".
* Else decision must be "fail".

OUTPUT CONSISTENCY CHECK:
Before returning the final JSON, check every result for consistency:

* If decision is "pass", then relevance_pass, answerability_pass, and query_quality_pass
  must all be true.
* If relevance_pass is false, decision must be "fail".
* If answerability_pass is false, decision must be "fail".
* If decision is "revise", then relevance_pass and answerability_pass must be true, and
  query_quality_pass must be false.
* Do not output contradictory combinations such as decision="pass" with relevance_pass=false.
* Do not include self-correction, uncertainty, chain-of-thought, or meta-commentary in
  failure_reason.
* failure_reason must be a concise final judgment only.

FAILURE REASON RULES:

* If Relevance fails, explain briefly why the query is not about emotion.
* If Answerability fails, explain briefly why the moment is unsupported, mismatched,
  hallucinated, unclear, or not reliably locatable.
* If Query Quality fails or needs revision, explain the wording issue briefly.
* Do not write long explanations.
* Do not include phrases such as "Wait", "Actually", "I must follow instructions", or any
  self-correction.

SUGGESTED REVISION RULES:

* If decision is "pass", suggested_revision must be an empty string.
* If decision is "revise", provide a concrete improved version of the query.
* If decision is "fail" because Relevance or Answerability fails, suggested_revision should
  usually be an empty string unless a simple rewrite can make the same observed moment
  suitable.
* Do not invent new video content in suggested_revision.
