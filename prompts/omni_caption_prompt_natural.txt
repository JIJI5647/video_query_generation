PROMPT VERSION: omni_caption_prompt_NATURAL_ablation

You are generating NATURAL-LANGUAGE multimodal captions for short video segments.

You are given one or more short video clips (with audio), each one temporal
segment of a longer video, provided IN ORDER. Watch and listen to every clip,
then caption EVERY clip — one caption per clip, no skipping and no merging.

CLIP -> SEGMENT MAPPING (the clips are provided in THIS order):
{segment_list}

The first video provided is Clip 1, the second is Clip 2, and so on. Use this
mapping to attach the correct segment_id and time_range to each caption.

Return ONLY a valid JSON array — one object per clip, in clip order. No markdown
fences, no extra text. Even for a single clip, return an array with one object.

Each caption object must follow this structure exactly (echo the segment_id and
time_range from the mapping). Put ALL of the visual description into ONE flowing
natural-language paragraph in "temporal_description" — describe the people (with
natural descriptions like "the woman in a white shirt"), their actions, the
scene/setting, notable objects and interactions, AND their observable facial
cues, body posture, gestures, and gaze — all as prose, not as fields. Leave the
structured fields empty:
{
  "segment_id": "<the clip's segment_id from the mapping>",
  "time_range": [<start>, <end>],
  "visual_objective": {"people": [], "scene": {"location": "", "setting": ""}, "objects": [], "interactions": [], "key_actions": [], "visibility_notes": ""},
  "visual_expression": [],
  "audio_description": "<one sentence on how the audio sounds, not the words>",
  "temporal_description": "<a rich natural-language paragraph: who is present, what they do, the scene, objects/interactions, and their observable facial expression, posture, gestures and gaze, and how it progresses across the clip>",
  "confidence": "<high | medium | low>",
  "evidence_strength": "<clear | ambiguous | weak>"
}

RULES (apply to every caption):

Describe only the current clip.

Put the WHOLE visual description in temporal_description as natural prose. Keep
visual_objective and visual_expression EMPTY (as shown above) — this is the
natural-language format.

Do not generate transcript text. audio_description describes how the audio
sounds (excited voice, shouting, crying, laughing, heavy breathing, silence,
unclear audio, off-screen voice), not the exact spoken words.

Do not use internal person IDs such as person_1 or person_2. Use natural
descriptions such as "the woman in a white shirt" or "the man in a suit". Do not
mention segment_id in natural-language fields.

Do NOT name or infer any emotion anywhere in the caption. This is an
observation-only caption: describe what is visible and audible (including facial
cues, posture, gestures, gaze), never what the person feels. Emotion is decided
in a later, separate stage. If a cue is not clearly visible or audible, say so in
the prose and set evidence_strength to "weak"/"ambiguous".

confidence must be one of: high | medium | low
evidence_strength must be one of: clear | ambiguous | weak

Output exactly one object per clip, in clip order, as a JSON array:
[
  { "segment_id": "...", "time_range": [..], ... },
  ...
]
