PROMPT VERSION: omni_caption_prompt_v1

You are generating a structured multimodal caption for one short video segment.

You are given exactly ONE short video clip (with its audio). It is a single
temporal segment of a longer video. Watch and listen to this clip only and
describe THIS segment.

SEGMENT METADATA (use only as metadata, do NOT mention it in any natural-language field):
  segment_id: {segment_id}
  time_range: [{start_time}, {end_time}]

Return only valid JSON. Do not include markdown fences or extra explanation.

The caption must describe the current segment only.

Do not generate transcript text. Audio description should describe how the audio
sounds, not the exact spoken words.

Do not use internal person IDs such as person_1 or person_2. Use natural
descriptions such as "the woman in a white shirt" or "the man in a suit".

Keep segment_id only as metadata. Do not mention segment_id in natural-language fields.

visual_objective must describe only objective visual facts: people, actions,
scene, objects, interactions, positions, and visibility. Do not include emotion
words here.

visual_expression must describe observable facial cues, body cues, posture,
gestures, and gaze (visible affective cues only — no interpretation).

audio_description must describe non-transcript audio information, such as excited
voice, tense voice, shouting, crying, laughing, heavy breathing, silence,
unclear audio, or off-screen voice. It is one natural-language sentence, not a
transcript.

emotion_description is a candidate interpretation, not ground truth. It must be
supported by visual_expression or audio_description. It may mention multiple
people and may state uncertainty. If emotion is unclear, say it is unclear.

If evidence is weak or ambiguous, set evidence_strength to "weak" or "ambiguous"
and do not force a strong emotion label. If a cue is not clearly visible or
audible, write "unclear", "not clearly visible", or "not audible".

confidence must be one of: high | medium | low
evidence_strength must be one of: clear | ambiguous | weak

Output JSON must match this structure exactly (echo the segment_id and
time_range given above):
{
  "segment_id": "{segment_id}",
  "time_range": [{start_time}, {end_time}],
  "visual_objective": {
    "people": [
      {
        "person": "<natural description, e.g. a woman in a white shirt>",
        "visibility": "<clearly visible | partially visible | not clearly visible>",
        "position": "<foreground | background | left | right | ...>",
        "action": "<observable action>"
      }
    ],
    "scene": {
      "location": "<indoor room | street | ...>",
      "setting": "<short description of the setting>"
    },
    "objects": [],
    "interactions": [],
    "key_actions": [],
    "visibility_notes": ""
  },
  "visual_expression": [
    {
      "person": "<the same natural description>",
      "facial_cues": ["<observed facial cue>"],
      "body_cues": [],
      "gaze": "<where they are looking>"
    }
  ],
  "audio_description": "<one sentence on how the audio sounds, not the words>",
  "emotion_description": "<candidate emotion reading, supported by cues above>",
  "confidence": "<high | medium | low>",
  "evidence_strength": "<clear | ambiguous | weak>"
}
